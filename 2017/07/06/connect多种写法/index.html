connect多种写法

星期四, 七月 6日 2017, 10:04 晚上

最近很长一段时间都在啃redux，看完阮一峰的redux入门又看官方文档，似懂非懂，上Github查看项目案例，其中的connect写法就很多种，让我难受的很。直到在啃《React与Redux开发实例精解》时，看到了connect的多种写法，才算是解了惑。

在学习redux的道路上，虽然难受兼坎坷，但是感觉还是收获满满。比如在看官方文档的态度上，以前面对官方文档总是害怕与逃避，冷静不了啃下去。这次的redux上，学习资料还不是很丰富，所以听从了前辈的建议，从官方文档入手，再配合一些书籍或精彩博文等等。

以下介绍connect的多种写法，例子依然是官方的计数器Demo。

一.

1.``connect()``的第一个参数是参数为``state``的函数，该函数返回的对象将被合并到组件的``props``中。``connect()``第二个参数是多个``action creator``组成的对象，该对象也会被合并到组件的``props``中，而且可以直接运行，无须使用``dispatch``。

import Counter from '../components/Counter';
import { connect } from 'react-redux';
import * as ActionCreators from '../actions';

export default connect (
    state => ({ couter: state.counter }),
    ActionCreators
)(Counter);
